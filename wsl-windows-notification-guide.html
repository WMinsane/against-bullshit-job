1. 全体像 — 何が起きているのか

WSL2 (Linux)
→
powershell.exe
→
WinRT Toast API
→
Windows 通知センター

WSL2 から Windows 側の powershell.exe を直接呼び出せる（相互運用機能）

ポイント: WSL2 は Linux カーネルだが、Windows のプロセスを直接起動できる。powershell.exe を叩けば Windows 側の API にアクセス可能。つまり Linux のシェルスクリプトから Windows のトースト通知を飛ばせる。

WSL2 の相互運用
WSL2 では /mnt/c/ 経由で Windows ファイルシステムにアクセスでき、*.exe を直接実行できる。逆に Windows 側から wsl コマンドで Linux を呼ぶこともできる。この双方向ブリッジが通知連携の土台。

2. 前提知識 — Windows 通知の仕組み

Windows 10/11 には大きく2種類の通知メカニズムがある:

方式API音の制御通知センター残留
バルーン通知（レガシー）NotifyIcon.ShowBalloonTip()ToolTipIcon で種類を選ぶだけ。None でも OS 設定次第で鳴る一時的
トースト通知（モダン）Windows.UI.Notifications (WinRT)<audio silent='true'/> で完全無音残る
落とし穴: NotifyIcon の ToolTipIcon.None は「アイコンなし」の意味であって「無音」ではない。OS の通知音設定に従うため、結局音が鳴る。これが最初のアプローチで無音にできなかった理由。
WinRT (Windows Runtime) とは
Windows 8 以降に導入されたモダン API 群。UWP アプリ向けだが、PowerShell からも [Windows.UI.Notifications.ToastNotificationManager] のように .NET 経由でアクセスできる。トースト通知の XML スキーマで <audio> 要素を制御でき、ここで silent='true' を指定すると確実に無音になる。

3. 方法1: NotifyIcon（簡単だが音が消せない）

参考として載せるが、無音が必要なら方法2を使うべき。

# WSL から実行
powershell.exe -Command "
  Add-Type -AssemblyName System.Windows.Forms
  \$n = New-Object System.Windows.Forms.NotifyIcon
  \$n.Icon = [System.Drawing.SystemIcons]::Information
  \$n.Visible = \$true
  \$n.ShowBalloonTip(3000, 'タイトル', '本文', 'None')
  Start-Sleep -Seconds 4
  \$n.Dispose()
"

仕組み

  1. System.Windows.Forms をロード（.NET のデスクトップ UI ライブラリ）
  2. NotifyIcon = タスクトレイのアイコン。これがないとバルーン通知を出せない
  3. ShowBalloonTip(ミリ秒, タイトル, 本文, アイコン種類) でバルーンを表示
  4. Dispose() でトレイアイコンを片付ける（やらないとゴーストアイコンが残る）
問題点: 第4引数 None はアイコン種類が「なし」になるだけ。通知音は OS の「通知」サウンド設定に従って鳴る。レジストリをいじって一時的に消す方法もあるが、レースコンディションがあり確実ではない。

4. 方法2: WinRT Toast API（無音対応・本命）

最小構成（これだけコピーすれば動く）

# WSL から実行
powershell.exe -Command "
[Windows.UI.Notifications.ToastNotificationManager, Windows.UI.Notifications, ContentType = WindowsRuntime] | Out-Null
[Windows.Data.Xml.Dom.XmlDocument, Windows.Data.Xml.Dom, ContentType = WindowsRuntime] | Out-Null

\$xml = @'
<toast>
  <visual>
    <binding template='ToastGeneric'>
      <text>タイトル</text>
      <text>本文メッセージ</text>
    </binding>
  </visual>
  <audio silent='true'/>
</toast>
'@

\$doc = New-Object Windows.Data.Xml.Dom.XmlDocument
\$doc.LoadXml(\$xml)
\$toast = [Windows.UI.Notifications.ToastNotification]::new(\$doc)
\$appId = '{1AC14E77-02E7-4E5D-B744-2EB1AE5198B7}\WindowsPowerShell\v1.0\powershell.exe'
[Windows.UI.Notifications.ToastNotificationManager]::CreateToastNotifier(\$appId).Show(\$toast)
"

各行の意味

行やっていること
[Windows.UI.Notifications...] | Out-NullWinRT の型をPowerShellセッションにロード。Out-Null は出力を捨てるだけ
$xml = @'...'@トースト通知の定義XML（ヒアストリング構文）
<audio silent='true'/>これが無音のキモ。この1行で通知音を完全に抑制
XmlDocument → LoadXmlXML文字列をDOMオブジェクトにパース
ToastNotification::new($doc)DOMから通知オブジェクトを生成
CreateToastNotifier($appId)通知を発行するアプリを指定（後述）
.Show($toast)通知を表示!

トースト XML のスキーマ

トースト通知の見た目と挙動はすべて XML で定義する。Microsoft のトーストスキーマに従う。主な要素: 
<toast>                          <!-- ルート要素 -->
  <visual>                       <!-- 見た目の定義 -->
    <binding template='ToastGeneric'>
      <text>1行目（太字タイトル）</text>
      <text>2行目（本文）</text>
      <text>3行目（補足、小さめ）</text>
    </binding>
  </visual>
  <audio silent='true'/>         <!-- 無音にする。省略すると既定音が鳴る -->
  <!-- <audio src='ms-winsoundevent:Notification.Default'/>  音を鳴らす場合 -->
</toast>

5. なぜ AppID が必要なのか

Windows のトースト通知は「どのアプリが出したか」を必ず紐づける。UWP アプリなら自動的に設定されるが、PowerShell スクリプトから出す場合は手動で既存アプリの AppID を借りる必要がある。

# PowerShell 自身の AppID（これを使うのが最も安全）
{1AC14E77-02E7-4E5D-B744-2EB1AE5198B7}\WindowsPowerShell\v1.0\powershell.exe

この文字列は PowerShell の「AppUserModelID」で、Windows のスタートメニューに登録されたアプリの識別子。通知センターではこの ID に基づいて「Windows PowerShell からの通知」と表示される。

AppID を省略 or 適当な文字列にすると: 通知が表示されないか、通知センターに残らない。先ほど 'Claude Code' という架空の AppID を指定して通知が消えたのはこれが原因。
AppID を調べる方法
Get-StartApps コマンドで Windows に登録されたアプリ一覧と AppID が見れる。カスタム AppID を登録する方法もあるがレジストリ操作が必要で、通常は PowerShell の AppID を借りるのが手軽。

6. シェルスクリプト化して使い回す

汎用通知スクリプト

#!/bin/bash
# notify.sh — WSL から Windows 無音トースト通知を送る
# 使い方: ./notify.sh "タイトル" "メッセージ"

TITLE="${1:-通知}"
MSG="${2:-完了しました}"

powershell.exe -Command "
[Windows.UI.Notifications.ToastNotificationManager, Windows.UI.Notifications, ContentType = WindowsRuntime] | Out-Null
[Windows.Data.Xml.Dom.XmlDocument, Windows.Data.Xml.Dom, ContentType = WindowsRuntime] | Out-Null

\$xml = @\"
<toast>
  <visual>
    <binding template='ToastGeneric'>
      <text>$TITLE</text>
      <text>$MSG</text>
    </binding>
  </visual>
  <audio silent='true'/>
</toast>
\"@

\\\$doc = New-Object Windows.Data.Xml.Dom.XmlDocument
\\\$doc.LoadXml(\\\$xml)
\\\$toast = [Windows.UI.Notifications.ToastNotification]::new(\\\$doc)
\\\$appId = '{1AC14E77-02E7-4E5D-B744-2EB1AE5198B7}\WindowsPowerShell\v1.0\powershell.exe'
[Windows.UI.Notifications.ToastNotificationManager]::CreateToastNotifier(\\\$appId).Show(\\\$toast)
"
# 使い方
chmod +x notify.sh
./notify.sh "Claude Code" "ビルド完了"
./notify.sh "テスト" "全件パス ✓"

7. Claude Code のフックに組み込む

Claude Code のフック（Hooks）とは
Claude Code はツール呼び出し等のイベント時にシェルコマンドを自動実行できる。.claude/settings.json に定義する。これを使えば「タスク完了時に自動で通知」が実現できる。

設定例: Notification Hook

// .claude/settings.json
{
  "hooks": {
    "notification": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "/path/to/notify.sh \"Claude Code\" \"$CLAUDE_NOTIFICATION_TITLE\""
          }
        ]
      }
    ]
  }
}

フックの仕組み

  1. Claude Code がバックグラウンドタスクを完了すると notification イベントが発火
  2. 設定されたコマンド（notify.sh）が自動実行される
  3. Windows 側にトースト通知が飛ぶ（無音）

これにより、指示を投げて別の作業をしていても、完了が分かる。

8. トラブルシューティング

症状原因対処
通知が表示されないAppID が無効PowerShell の GUID AppID を使う（上記参照）
音が鳴るNotifyIcon 方式を使っているWinRT Toast API に切り替える
powershell.exe が見つからないWSL の PATH に Windows パスが含まれていない/mnt/c/Windows/System32/WindowsPowerShell/v1.0/powershell.exe をフルパスで指定
WinRT 型のロードエラーPowerShell 5.1 以前 or .NET ランタイム不足pwsh.exe（PowerShell 7）は WinRT 非対応の場合あり。powershell.exe（5.1）を使う
通知が一瞬で消えるWindows の集中モード / 通知設定設定 → システム → 通知 → PowerShell の通知を許可
XML パースエラーヒアストリング内の変数展開 or エスケープPowerShell の @'...'@（シングルクォート）は変数展開しない。Bash 経由の場合はエスケープに注意
まとめ: WSL → Windows 通知は「powershell.exe で WinRT Toast API を叩く」のが正解。<audio silent='true'/> で確実に無音にでき、AppID に PowerShell の GUID を使えば通知センターにも残る。シェルスクリプト化すれば任意の CLI ツールから ./notify.sh "タイトル" "完了" で呼べる。
